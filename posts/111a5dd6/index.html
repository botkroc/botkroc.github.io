寄存器

通用寄存器可以储存任何数据和指令

32位 16位 8位
EAX AX AL
ECX CX CL
EDX DX DL
EBX BX BL
ESP SP AH
EBP BP CH
ESL SL DH
EDI DI BH

16位寄存器使用对应32位寄存器的低十六位

8位寄存器使用32位寄存器的两个字节 L为低八位，H为高八位

假设EAX当前为0XFFFFFFFF,执行第一行指令后变为0XFFFF01FF，执行第二行指令后变为0XFFFF0102

地址值的五种形式

  1. 立即数

    读取内存的值：
    MOV EAX,DWORD PTR DS:[0x13FFC4]

    向内存写入数据：
    MOV DWORD PTR DS:[0X13FFC4],EAX

  2. [reg]

    读取内存的值：
    MOV ECX,0X13FFD0
    MOV EAX,DWORD PTR DS:[ECX]

    向内存写入数据：
    MOV EDX,0X13FFD8
    MOV DWORD PTR DS:[EDX],0X87654321

  3. [reg + 立即数]

    读取内存的值：
    MOV ECX,0X13FFD0
    MOV EAX,DWORD PTR DS:[ECX+4]

    向内存写入数据：
    MOV EDX,0X13FFD8
    MOV DWORD PTR DS:[EDX+0XC],0X87654321

  4. [reg + reg * (1,2,4,8)]

    多用于高级语言中声明数组

    读取内存的值：
    MOV EAX,13FFC4
    MOV ECX,2
    MOV EDX,DWORD PTR DS:[EAX+ECX*4]

    向内存写入数据：
    MOV EAX,13FFC4
    MOV ECX,2
    MOV DWORD PTR DS:[EAX+ECX*4],87654321

  5. [reg + reg * (1,2,4,8) + 立即数]

    读取内存的值：
    MOV EAX,13FFC4
    MOV ECX,2
    MOV EDX,DWORD PTR DS:[EAX+ECX*4+4]

    向内存写入数据：
    MOV EAX,13FFC4
    MOV ECX,2
    MOV DWORD PTR DS:[EAX+ECX*4+4],87654321

储存模式

有两种储存模式

  1. 大端模式：数据高位在低位，数据低位在高位

  2. 小端模式：数据低位在低位，数据高位在高位

对于数据来说，1A在高位，2C在低位，而对于内存来说0X00000000是低位，0XFFFFFFFF是高位

此时在大端模式中内存中储存情况是0X00000000的值为0X1A，0X00000001的值为0X2C。小端模式则相反，0X2C（也就是低位）储存在内存中的低位

常用汇编指令

内存相关指令

MOV BYTE PTR DS:[0018FFF0],1

将立即数储存到0X0018FFF0开始的一字节空间，在写入内存时必须声明长度。

在汇编中的数据类型有BIT（1位），BYTE（1字节），WORD（2字节），DWORD（4字节）

MOV DWORD PTR DS:[0019FF74],EAX
将32位通用寄存器EAX的值写到0X0019FF74，写入的宽度必须和寄存器的宽度一致

MOVS指令

MOV 指令无法从内存写到内存

32位CPU有2个32位通用寄存器ESI和EDI。其低16位对应先前CPU中的SI和DI，对低16位数据的存取，不影响高16位的数据。

寄存器ESI、EDI、SI和DI称为变址寄存器(Index Register)**，它们主要用于存放存储单元在段内的偏移量，用它们可实现多种存储器操作数的寻址方式，为以不同的地址形式访问存储单元提供方便。变址寄存器不可分割成8位寄存器。**

可以简写成MOVSB，MOVSW，MOVSD，ES是extra segment的缩写，执行以上命令，会将寄存器ESI中指向内存的数据储存到EDI指向的内存中，执行完毕后EDI和ESI两个寄存器中的地址会自动增加，增加的数取决于附加数据的长度，以上三条指令分别会使变址寄存器中的值增加1、2、4

在使用时EDI必须作为储存附加段地址寄存器，ESI作为储存数据段地址的寄存器

假如有以下命令

MOVS DWORD PTR ES:[EDI],DWORD PTR DS:[ESI]

执行完毕后两个变址寄存器的值增加了4，因为写入了4字节数据，内存往后移4字节。
