DHCP部署与安全

发表2020-12-15|更新2020-12-30|渗透测试
  • DHCP原理
  • DHCP续约
  • 命令
  • DHCP攻击与防御

  • DHCP
    Dynamic Host Configuration Protocol

  • DHCP相关概念
    地址池/作用域：（IP、子网掩码、网关、DNS、租期），DHCP使用UDP协议

  • DHCP优点
    减少工作量、避免IP冲突、提高地址利用率

    • 在cisco路由器上面配置DHCP
      ip helper-address 配置DHCP服务器的地址(不需要子网掩码)，此路由器成为DHCP中继

DHCP原理

成为DHCP租约过程，分为4个步骤：

  1. 服务器发送DHCP Discovery广播包
    客户机广播请求IP地址（包括客户机的MAC地址）

  2. 客户机响应DHCP Offer广播包
    服务器响应提供的IP地址（但无子网掩码、网关等参数）

  3. 服务器发送DHCP Request广播包
    客户机选择IP

  4. 服务器发送DHCP ACK广播包(Acknowledgement)
    服务器确定了租约，并提供网卡详细参数IP、掩码、网关、DNS、租约

DHCP续约

当50%过后，客户机会再次发送DHCP Resuqst包，进行续约，如服务器无响应，则继续使用并在87.5%再次发送DHCP Resuqst包，如仍然无响应，则释放IP地址，重新发送DHCP Discovery广播包。

当无任何服务器响应时，在windows系统下，网卡会自动分配169.254.X.X/16的无效地址，保证内网临时通讯。

在vmnet1中自带了DHCP服务器，用vmnet2或者将vmnet1的DHCP关闭

VM中vmnet1和vmnet8自带DHCP服务

命令

ipconfig /renew
重新请求ip地址

ipconfig /release
释放租约

DHCP攻击与防御

  • 攻击DHCP服务器

    频繁地发送伪装DHCP请求，直到将DHCP地址池资源耗尽

    防御：在交换机（管理型）的端口上做动态MAC地址绑定

  • 伪装DHCP服务器攻击：

    hack通过将自己部署为DHCP服务器，为客户机提供非法ip地址

    防御：企业级交换机通过限制DHCP Offer广播包来防止伪造DHCP服务器，在交换机上，除合法的DHCP服务器所在接口外，全部禁止发送DHCP Offer包
