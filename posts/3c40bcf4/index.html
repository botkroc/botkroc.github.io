域名系统 服务

协议(Domain Name System)

以下情况需要安装自己的DNS服务器

  1. 解析内网网站域名
  2. 降低到Internet的域名解析流量
  3. 域环境

    机器在处理IP数据报时使用IP地址而不使用域名的原因：因为IP地址的长度是固定的32位（如果是IPv6则是128位，也是固定的），而域名的长度不是固定的，机器处理起来很麻烦。

域名解析过程

域名到IP地址的解析过程是由分布在互联网上的许多域名服务器共同完成的。当某个应用程序需要把主机名解析成IP地址时，该应用程序就调用解析程序，并成为DNS的一个客户，把待解析的域名放在一个DNS请求报文中，以UDP用户数据报方式发送给本地域名服务器（指物联网上的DNS服务器，使用UDP协议是为了减少开销）。本地域名服务器在找到域名后，把对应的IP地址放在回答报文返回。应用程序获得目的主机的IP地址后即可进行通信。
如果本地域名服务器不能解析DNS，则此域名服务器暂时成为DNS中的另一个客户，并向其他域名服务器发出查询请求。这种过程直到找到能够回答请求的域名服务器为止。

域名系统的树状结构：

实际域名系统

DNS服务器的管辖范围不是以域为单位而是以区为单位，区是DNS服务器的实际管辖范围，
区可能小于等于域，但不能大于域。每一个区设置一个权限域名服务器（authoritative name server），
用来保存该区中所有主机的域名到IP地址的映射。

根域名服务器 根域名服务器
顶级域名服务器 org域名服务器 com域名服务器 edu域名服务器 …
权限域名服务器 abc.com域名服务器
y.abc.com域名服务器 （abc公司有两个权限域名服务器）

根域名服务器是最高层次的域名服务器，所有的根域名服务器都知道所有的顶级域名服务器的域名和IP地址。

不管是哪一个本地域名服务器，若要对互联网上的任何一个域名进行解析，只要自己无法解析，就首先求助于根域名服务器。假设所有的根域名服务器都瘫痪了，那么整个互联网中的DNS系统都无法正常工作。根域名服务器并不直接把待查询的域名直接转换成IP地址（根域名服务器也没有存放这种信息），而是告诉本地服务器下一步应当找哪一个顶级域名服务器进行查询。
用nslookup查询DNS服务器非权威是指本地域名服务器上没有要查找主机的缓存。

DHCP动态主机配置协议(dynamic host configuration protocol)

windows命令行

释放租约命令 ipconfig /release
重新获得地址 ipconfig /renew
路由器会隔绝广播包，如果要跨网段分配IP地址，
需要在路由器上配置IP heaper address(DHCP服务器的IP地址，此路由器成为DHCP中继代理)，
需要先在DHCP服务器添加要分配IP地址的网段的网关，然后在路由器上使用ip helper-address分配DHCP服务器的IP地址(不需要子网掩码)

FTP文件传输协议(File Transfer protocol):
FTP的主要功能是消除不同操作系统下处理文件的不兼容型

FTP连接方式

控制连接：标准端口为21，用于发送FTP命令信息。
数据连接：标准端口为20，用于上传下载数据。

数据连接的建立类型

主动模式

服务器从20端口主动向客户端发起连接

被动模式

服务器在指定范围内的某个端口被动等待客户端发起连接。
使用主动还是被动取决于客户端的设置，如果FTP服务器只开20和21端口，则无法使用被动模式访问。
FTP服务器进程由两大部分组成：一个主进程，负责接受新的请求。另外有若干从属进程，负责处理单个请求。
在进行文件传输时，FTP的客户端和服务器之间建立两个并行的TCP连接：控制连接和数据连接。
控制连接在整个会话期间一直保持打开，实际用于传输的是数据连接。
FTP服务器端，如果有防火墙，需要在防火墙打开20和21端口，使用主动模式进行数据连接。

telnet远程终端协议

telnet可以用来测试目标主机的指定端口是否打开

http

<协议>://<主机>:<端口>/<路径>
相同主机的多个网站可以使用不同的IP地址或不同的端口来区分
