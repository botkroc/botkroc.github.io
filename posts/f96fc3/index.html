PE文件的整体结构

PE文件所有结构体的定义都在winnt.h文件里，具体详见官方文档

IMAGE_DOS_HEADER(64)
IMAGE_FILE_HEADER(20)
IMAGE_OPTIONAL_HEADER32(224)
IMAGE_SECTION_HEADER(40)

DOS部分

IMAGE_DOS_HEADER结构占64字节，DOS Stub大小是不确定的，IMAGE_DOS_HEADER结构的最后一个成员e_lfanew是一个LONG类型的偏移，它标识了PE头从哪开始。MZ头+偏移+PE头标识就是PE指纹。IMAGE_DOS_HEADER和PE头中间的部分就是DOS Stub。除了首尾两个成员外其他的成员在32位和64位程序中不使用。

DOS Stub由链接器生成，在Windows系统中没有作用，一般输入内容是This program cannot be run in DOS mode

PE头

32位

64位

PE头由三部分组成：PE标识（占4字节）、标准PE头（IMAGE_FILE_HEADER占20字节）、拓展PE头（IMAGE_OPTIONAL_HEADER32大小为0xE0，224字节。64位中为0xF0，240字节），拓展PE头的大小可以在IMAGE_FILE_HEADER结构（标准PE头）中修改。

节表

节表是一个IMAGE_SECTION_HEADER结构类型的数组，每个成员占40个字节，IMAGE_FILE_HEADER.NumberOfSections标识了其数量

定位节表需要先确定拓展PE头的大小，IMAGE_FILE_HEADER.SizeOfOptionalHeader标识了拓展PE头的大小。

注意，Name成员并不要求遵守以\0结尾，最安全的做法是截取其中8个字节处理，否则会越界。

Misc之所以使用匿名联合主要是为了可读性。

Characteristics常用特征值：

标志位 含义
IMAGE_SCN_CNT_CODE
0x00000020		 The section contains executable code.
包含代码，常与 0x10000000一起设置。
IMAGE_SCN_CNT_INITIALIZED_DATA
0x00000040		 The section contains initialized data.
该区块包含以初始化的数据。
IMAGE_SCN_CNT_UNINITIALIZED_DATA
0x00000080		 The section contains uninitialized data.
该区块包含未初始化的数据。
IMAGE_SCN_MEM_DISCARDABLE
0x02000000		 The section can be discarded as needed.
该区块可被丢弃，因为当它一旦被装入后，进程就不在需要它了，典型的如重定位区块。
IMAGE_SCN_MEM_SHARED
0x10000000		 The section can be shared in memory.
该区块为共享区块。
IMAGE_SCN_MEM_EXECUTE
0x20000000		 The section can be executed as code.
该区块可以执行。通常当0x00000020被设置时候，该标志也被设置。
IMAGE_SCN_MEM_READ
0x40000000		 The section can be read.
该区块可读，可执行文件中的区块总是设置该标志。
IMAGE_SCN_MEM_WRITE
0x80000000		 The section can be written to.
该区块可写。

PE文件的两种状态

在拓展PE头中有一个成员IMAGE_OPTIONAL_HEADER32.SizeOfHeaders标识了所有头+节表的大小，大小按照拓展PE头中的另一成员FileAlignment对齐，节数据在磁盘上也按照FileAlignment对齐

如上图，在拓展PE结构往后偏移36个字节找到FileAlignment，再往后偏移20个字节找到SizeOfHeaders。SizeOfHeaders按照FileAlignment对齐，大小一定是其整数倍。此处到0x400为止，往后就是节数据了

IMAGE_OPTIONAL_HEADER32.SectionAlignment标识了PE文件在内存中时的对齐方式，在FileAlignment前面4个字节就是SectionAlignment

在内存中所有头+节表(SizeOfHeaders)也按照文件对齐，但会一直延伸到SectionAlignment的位置，节数据实际开始的位置一定是SectionAlignment的整数倍。整个PE镜像内存也按照SectionAlignment对齐
