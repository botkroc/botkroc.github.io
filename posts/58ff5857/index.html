使用Git将项目部署到远程仓库

命令

命令 描述
git init 初始化本地仓库 执行此命令时git会自动创建master分支
git config –global user.name “”

git config –global user.email “”		 提交用户信息
git add 添加指定或全部已修改文件到暂存区
git commit 将暂存区文件提交到本地仓库
git branch 没有参数时会列出所有分支
git branch -d branchname 删除分支
git checkout xxx 切换到指定分支
git remote add [shortname] [url] 添加远程仓库
git branch (branchname) 创建分支命令
git checkout (branchname) 切换分支命令,切换分支的时候，Git 会用该分支的最后提交的快照替换你的工作目录的内容
git ls-files 查看暂存区的文件
git push -u sample master 提交当前分支到sample下的master分支 如果不加参数则提交所有分支
git fetch 提取分支
git merge 合并到本地仓库
git pull 相当于以上2个命令
git pull origin master：brantest 将远程仓库origin的master分支拉去过来 与本地仓库brantest合并,如果要与本地的当前分支合并则可以省略冒号后的参数
git remote 查看远程仓库 -v 查看地址

基本使用方法

  • 不带任何参数的git push，默认只推送当前分支

  • 本地仓库下有个.gitignore 可以忽略特定的文件

  • 生成 SSH

  • git restore 命令不会删除已经存在的文件

提交项目时报错 git Filename too long

git有可以创建4096字节长度文件名，在windows系统下最多是260字节

解决方法：

在服务器上从远程仓库拉取项目

在确保user.name和user.email一样的情况下直接将本机.ssh文件夹下个两个文件复制到服务器主机

将本机的.git文件夹复制到服务器上，否则第一次拉取会失败。原因是版本库不一致

根目录下的db.json最好忽略，否则每次清除缓存都会改变这个文件

github有时速度很慢可以将项目同步到gitee 再使用git clone
